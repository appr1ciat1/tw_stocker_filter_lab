# 動態候選池研究流程（先研究、後投資）

> **最高原則**：符合真實投資人行為 — **先研究、後投資**。
> 資金**只**在最後階段投入；前面每一步都在「零資金」下完成。
> 階段閘門**只能順序前進，不可跳、不可倒序**；任一門未過就停在原地，生產線續用現行 116 檔。

本文件把前面建好的工具（`pool_generator` / `pool_acceptance` / `pool_audit` / `validation/*` /
`preflight` / `capacity`）串成一條有序的研究→投資管線，並固化門檻與鐵律。

---

## 一、研究 / 投資 的硬分界

```
════════════════ 研究區（零資金，可反覆、可放棄）════════════════
  Phase 0  資料前提        →  Phase 1  候選池建構(可調參)
                           →  Phase 2  樣本外三門驗收
                           →  Phase 3  影子/紙上追蹤(前瞻、仍零資金)
──────────────────  ⛔ 資金分界線（過此才動錢）──────────────────
════════════════ 投資區（真實資金，漸進、可回退）════════════════
                           →  Phase 4  同 run A/B 漸進部署 + 監控回退
```

**跨越分界線的唯一條件**：Phase 2 三門全過 **且** Phase 3 影子期行為與回測一致。
未達成 → 永遠停在研究區。這就是「先研究、後投資」。

## 二、逐階段（目的 / 工具 / 通過閘門 / 禁止事項）

### Phase 0 — 資料前提（研究）
- **目的**：備妥 point-in-time 全市場成交額歷史，凍結後不再變動（可重現）。
- **工具**：`pool_audit.fetch_market_history`（TWSE `MI_INDEX` + TPEx `dailyQuotes`，上市+上櫃）。
- **通過閘門**：資料契約 —— 每日完整度、關鍵標的、無異常跳變（沿用 `twstk/data/contract.py` 精神）。
- **禁止**：用任何「今天才知道」的資訊回填過去；此階段不做任何選股決策。

### Phase 1 — 候選池建構（研究，**唯一可調參處**）
- **目的**：在**訓練期**建構逐日 point-in-time 候選池，並在此決定參數。
- **工具**：`twstk/data/pool_generator.build_pointintime_pools`
  - trailing **中位數**成交額 `shift(1)` 嚴格排除當日（無前視）
  - 進出緩衝 `enter_rank=130 / exit_rank=170`（壓 churn）
  - 絕對地板 `min_adv`、最短上市天數 `min_history`
- **通過閘門**：無（這裡是探索）。但**所有調參止於此**。
- **禁止**：在 Phase 2 之後回頭改任何參數（否則 OOS 被污染，等於偷看答案）。

### Phase 2 — 樣本外三門驗收（研究）
- **目的**：在**訓練期沒看過的 OOS 區間**證明候選池「不是過擬合、買得到、且真的補上覆蓋」。
- **工具**：`pool_acceptance.run_acceptance`，三門**全過**才算通過：

  | 門 | 內容 | 通過門檻 | 重用 |
  |---|---|---|---|
  | ① 統計 | PBO/CSCV、Deflated Sharpe、nested walk-forward | PBO ≤ 0.5、DSR 機率 ≥ 0.95、逐年 OOS Sharpe ≥ v8.5 基準 | `validation/pbo_cscv.py`、`deflated_sharpe.py`、`walk_forward_nested.py` |
  | ② 容量 | 意圖部位 participation、出清天數 | participation ≤ ADV 10%、出清 ≤ 1 日 | `validation/capacity.py` |
  | ③ 覆蓋/腐化 | 對全市場 Top-150 覆蓋率、候選池自身腐化率 | 覆蓋 ≥ 80% **且** 腐化 ≤ 10% | `pool_audit.audit_pool` |

- **鐵律**：**沒驗證 ≠ 通過**（`run_acceptance` 缺輸入即記為 FAIL）。
- **禁止**：拿 in-sample 報酬冒充 OOS；跳過任一門。

### Phase 3 — 影子 / 紙上追蹤（研究，前瞻、**仍零資金**）
- **目的**：真實投資人「先紙上跑一段」——把候選池與現行 116 **並行**跑 N 個交易日，看**前瞻**表現，而非只有回測。
- **工具**：沿用 paper 追蹤基礎設施（`paper_tracker` / `build_paper_page`），候選池另開影子軌。
- **通過閘門**（建議）：影子期 ≥ 一段有意義期間（如 20–60 交易日）內，
  - 前瞻報酬/回撤與回測**同分布**（無 regime break）；
  - 實際換手/滑價 ≤ 容量門估計；
  - 追蹤誤差在容忍範圍。
- **禁止**：跳過影子期直接上錢；用影子期的好運回頭改參數。

### ⛔ 資金分界線

### Phase 4 — 漸進部署（投資，真實資金）
- **目的**：只有到這裡才動錢，且**漸進、可回退**。
- **鐵律**：
  1. **同 run 內 off vs on 全面改善才上**（沿用既有部署鐵律，見 [[v85-constrained-optimization]] 精神）。
  2. 漸進替換，現行 116 保留為 fallback；不一次全換。
  3. 生產四策略在此之前一律用現行 116。
  4. 設**自動回退**觸發（實盤指標劣於影子期門檻即回滾）。
- **禁止**：因為「回測很漂亮」跳過 Phase 3 直接全量部署。

## 三、順序鐵律（違反即代表「投資跑在研究前面」）

1. **Point-in-time**：第 t 日的池只用 ≤ t-1 資料（`pool_generator` 以 `shift(1)` 強制）。
2. **調參止於 Phase 1**：Phase 2 起不得回頭改參數。
3. **閘門單向**：Phase 2 全過才進 3；Phase 3 過才跨資金線進 4。不可跳、不可倒。
4. **生產隔離**：研究全程在研究分支/影子軌，碰不到生產四策略與 Paper 主軌。
5. **Fail-closed**：任一門「未驗證」都當 FAIL，不得預設通過。
6. **部署仍需同 run A/B**：Phase 4 不是自動放行，是再過一道 off vs on。

## 四、門檻總表（皆為可調風險旋鈕，此為目前確認值）

| 參數 | 值 | 出處 |
|---|---|---|
| 覆蓋率下限 | ≥ 80% | 本輪確認 |
| 腐化率上限 | ≤ 10% | 本輪確認 |
| PBO 上限 | ≤ 0.5 | 業界慣例 |
| Deflated Sharpe 機率 | ≥ 0.95 | 業界慣例 |
| 單檔 participation | ≤ ADV 10% | `capacity` 預設 |
| 出清天數 | ≤ 1 日 | `capacity` 預設 |
| 絕對流動性地板 | 中位數日成交額 ≥ NT$5,000 萬 | 審計方法論 |
| 進榜 / 續留 | Top-130 / Top-170 | hysteresis |
| 流動性窗 | 20 日中位數 | MSCI「中位數排除爆量」 |
